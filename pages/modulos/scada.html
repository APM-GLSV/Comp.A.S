SCADA

Para este proyecto se usó el software Ignition Gateway para la creación del sistema tipo SCADA, esta elección se realiza con base a la arquitectura planeada para el módulo Industria 4.0 y Comunicaciones ya que al ver que Ignition puede cumplir las funciones de Servidor MQTT y de Servidor SCADA nos parece adecuado tener todo centralizado en un mismo servicio.

El proyecto se divide en dos módulos, el primer módulo es el de almacenamiento el cual esta compuesto por una banda transportadora, dos sensores ópticos y el gantry, esta etapa se encarga de cargar las cajas a la bodega para mantener el almacén por encima del 30% durante toda la jornada de trabajo, esta etapa tiene accionamiento manual por medio del SCADA y desde allí se puede monitorear todo el proceso de alimentación de la bodega junto con el estado actual de cada tipo de cajas y su historial de uso.

Así vamos a tener un SCADA alojado en la nube con 4 vistas principales, en la primera encontraremos la vista general del estado de planta, en esta podremos dar la autorización para comenzar la operación del sistema y ver cuando este reciba la orden y nos habilite el modo AUTO.

Continuando con el recorrido en nuestro SCADA se puede ver el modulo de almacenamiento, este modulo cuenta con la visualización de sensores, estado de banda y cantidad de cajas por tipo junto con su histórico de consumo.

En la siguiente vista podremos ver el estado de los sensores la banda y el robot del módulo de distribución adicionalmente tenemos el selector de casos que nos permite indicar que caso de prueba vamos a ejecutar y posteriormente dar clic al botón que habilita el inicio de la operación.

Por último, tenemos nuestra vista de alarmas en estas podremos ver cuando es necesario llenar el almacén nuevamente ya que si algún tipo de caja baja a menos del 30% se dispara la alarma para que el operario comience el ciclo de llenado de caja antes de iniciar el siguiente ciclo.
